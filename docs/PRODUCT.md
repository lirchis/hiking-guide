# 产品草案

## 目标

Hiking Guide 希望解决的不是“列出很多路线”，而是把分散、口径不一、难以核验的徒步信息整理成可搜索和可比较的数据。

## 首版用户任务

1. 按地区、名称或标签找到候选路线。
2. 比较距离、累计爬升、预计时长和难度。
3. 了解路线类型、适合季节和基础风险提示。
4. 打开可信来源，进一步确认交通、天气、封闭和补给信息。

## MVP 范围

- 路线列表与详情
- 关键词、地区、难度和距离筛选
- 统一路线数据模型
- 来源与更新时间展示
- 地图起点和 GPX 轨迹展示

## 当前里程碑：50 / 100

第一阶段已完成可用的纯静态目录：

- 收录广东及周边 32 条路线、全国精选 18 条路线。
- 支持关键词、区域、难度和当前状态筛选。
- 支持路线质量、景观、难度和距离排序。
- 明确区分路线质量与 `available / verify / reference / closed` 当前状态。
- 使用相对路径和 GitHub Actions，可直接发布到 GitHub Pages 项目子路径。

地图、GPX、海拔剖面和剩余 50 条路线属于下一里程碑。

## 暂不纳入

- 实时天气与救援能力
- 未经审核的公开投稿自动上线
- 将难度标签替代个人风险判断
- 在未确认许可时复制商业平台路线或 GPX 数据

## 内容工作流

```text
候选路线 → 来源核验 → 数据录入 → 内容审核 → 发布 → 定期复核
```

每条路线至少要有一个明确来源和最近核验日期。涉及季节封闭、预约、收费、交通或危险路段的信息，应单独标记，后续接入更短的复核周期。

## 建议迭代顺序

### 阶段一：可用的数据目录

- 确定首个区域
- 收录 10–20 条种子路线
- 完成列表、筛选与详情页

### 阶段二：空间信息

- 接入开源底图
- 展示起点、轨迹和海拔剖面
- 支持 GPX 导入与下载

### 阶段三：协作维护

- 路线投稿与审核
- 变更记录和信息过期提醒
- 收藏、清单和行程计划
